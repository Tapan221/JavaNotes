### Q1: What is Java?
**A:** A high-level, object-oriented, platform-independent programming language created by James Gosling at Sun Microsystems in 1995 (now Oracle).

### Q2: What are the main features of Java?
**A:** Simple, Object-Oriented, Platform Independent (WORA), Secure, Robust, Multithreaded, Distributed, Portable, Dynamic, Architecture Neutral.

### Q3: What does "Write Once, Run Anywhere" mean?
**A:** Java compiles to bytecode (.class) which runs on any OS with a JVM. The bytecode is platform-independent.

### Q4: Difference between JDK, JRE, and JVM?
**A:**
- **JVM** → Executes bytecode (platform-dependent)
- **JRE** → JVM + libraries (to run programs)
- **JDK** → JRE + compiler + dev tools (to develop programs)

### Q5: What is JVM? Is it platform-independent?
**A:** JVM (Java Virtual Machine) executes bytecode. JVM itself is **platform-dependent** (different JVM for Windows, Linux, Mac), but the bytecode it runs is platform-independent.

### Q6: What is bytecode?
**A:** Intermediate code generated by `javac` compiler from `.java` file. Stored in `.class` file. Executed by JVM.

### Q7: What is the JIT compiler?
**A:** Just-In-Time compiler converts bytecode to native machine code at runtime for better performance. Part of JVM.

### Q8: What are the memory areas allocated by JVM?
**A:**
- **Heap** → Objects and instance variables
- **Stack** → Method calls, local variables, references
- **Method Area** → Class metadata, static variables
- **PC Register** → Current instruction address per thread
- **Native Method Stack** → For native (C/C++) method calls

### Q9: What is the difference between Heap and Stack memory?
**A:**
| Feature | Stack | Heap |
|---------|-------|------|
| Stores | Local variables, method calls | Objects, instance variables |
| Speed | Faster | Slower |
| Size | Small (per thread) | Large (shared) |
| Thread-safe | Yes (per thread) | No (shared) |
| Error | StackOverflowError | OutOfMemoryError |

### Q10: What is a ClassLoader?
**A:** Part of JVM that loads `.class` files into memory. Three types:
- Bootstrap ClassLoader → core Java classes (rt.jar)
- Extension ClassLoader → ext directory classes
- Application ClassLoader → classpath classes

### Q11: Explain the execution flow of a Java program.
**A:** Write `.java` → Compile with `javac` → Produces `.class` (bytecode) → ClassLoader loads into JVM → Bytecode Verifier checks → JIT compiles to native code → Executes.

### Q12: What is the `main()` method signature?
**A:** `public static void main(String[] args)`
- `public` → accessible by JVM from anywhere
- `static` → no object needed
- `void` → returns nothing
- `String[] args` → command-line arguments

### Q13: Can we run Java without `main()`?
**A:** No (since Java 7). Before Java 7, a `static` block could run code without `main()`, but not anymore.

### Q14: Can we overload `main()`?
**A:** Yes, but JVM always calls `public static void main(String[] args)`.

### Q15: Can we declare `main()` as `final`?
**A:** Yes. `public static final void main(String[] args)` is valid.

### Q16: Can we declare `main()` as `synchronized`?
**A:** Yes. It's valid but rarely needed.

### Q17: What happens if we write `static public void` instead of `public static void`?
**A:** It compiles and runs fine. Order of modifiers doesn't matter.

### Q18: What is a package in Java?
**A:** A namespace for organizing classes. Prevents naming conflicts. Example: `java.util`, `java.io`, `com.myapp.models`.

### Q19: What is `import` statement? What is static import?
**A:**
```java
import java.util.ArrayList;         // regular import
import java.util.*;                  // wildcard import
import static java.lang.Math.PI;    // static import → use PI directly
import static java.lang.Math.*;     // import all static members
```

### Q20: What is the default package?
**A:** When no package is specified, the class belongs to the **default package**. Classes in default package can't be imported by other packages.

### Q21: What is `System.out.println()`?
**A:**
- `System` → final class in `java.lang`
- `out` → static `PrintStream` object
- `println()` → method that prints and adds newline

### Q22: Difference between `println()`, `print()`, and `printf()`?
**A:**
```java
System.out.println("Hello");   // prints + new line
System.out.print("Hello");     // prints, NO new line
System.out.printf("Age: %d", 25); // formatted output
```

### Q23: What are comments in Java?
**A:**
```java
// single line comment
/* multi-line comment */
/** Javadoc comment — for documentation */
```

### Q24: What is `javadoc`?
**A:** A tool that generates HTML documentation from `/** */` comments in source code.

### Q25: What is Unicode?
**A:** A universal character encoding standard. Java uses **UTF-16**. `char` is 2 bytes to support Unicode.

## DATA TYPES & VARIABLES

### Q26: What are the 8 primitive data types?
**A:**
| Type | Size | Default | Range |
|------|------|---------|-------|
| `byte` | 1B | 0 | -128 to 127 |
| `short` | 2B | 0 | -32768 to 32767 |
| `int` | 4B | 0 | -2^31 to 2^31-1 |
| `long` | 8B | 0L | -2^63 to 2^63-1 |
| `float` | 4B | 0.0f | ~7 decimal digits |
| `double` | 8B | 0.0d | ~15 decimal digits |
| `char` | 2B | '\u0000' | 0 to 65535 |
| `boolean` | ~1bit | false | true/false |

### Q27: Why is `char` 2 bytes in Java?
**A:** Java uses **UTF-16** Unicode encoding, which requires 2 bytes to represent characters from all languages.

### Q28: What is the difference between `float` and `double`?
**A:** `float` = 4 bytes, ~7 digits precision, suffix `f`. `double` = 8 bytes, ~15 digits precision, default for decimals.

### Q29: Why can't we use `float f = 3.14;`?
**A:** `3.14` is a `double` literal by default. Must write `float f = 3.14f;` or cast: `float f = (float) 3.14;`.

### Q30: What are the types of variables?
**A:**
```java
class Demo {
    int x = 10;              // Instance variable (per object)
    static int y = 20;       // Static/Class variable (shared)
    void method() {
        int z = 30;          // Local variable (method only)
    }
}
```

### Q31: What is the default value of local variables?
**A:** **No default value.** Must be initialized before use, or compiler throws an error.

### Q32: What is the default value of instance variables?
**A:** `int` → 0, `double` → 0.0, `boolean` → false, `char` → '\u0000', Object → `null`.

### Q33: What is a constant in Java?
**A:** A variable declared with `final` keyword. Value can't be changed after initialization.
```java
final double PI = 3.14159;
// PI = 3.14;  → ERROR
```

### Q34: What is var keyword (Java 10)?
**A:** Local variable type inference — compiler infers type automatically.
```java
var name = "Rahul";    // inferred as String
var list = new ArrayList<String>(); // inferred as ArrayList<String>
// var only works for local variables, not fields or parameters
```

### Q35: What is type casting?
**A:**
```java
// Widening (Auto): small → large
int i = 100;
double d = i;  // 100.0

// Narrowing (Manual): large → small
double price = 99.99;
int rounded = (int) price;  // 99 (truncated)
```

### Q36: What is implicit vs explicit casting?
**A:** Implicit = automatic (widening, safe). Explicit = manual with `(type)` (narrowing, may lose data).

### Q37: What is the widening order?
**A:** `byte → short → int → long → float → double`. Also `char → int`.

### Q38: Can we cast `boolean` to `int`?
**A:** No. `boolean` cannot be cast to any other type in Java.

### Q39: What is type promotion in expressions?
**A:** In expressions, `byte`, `short`, `char` are automatically promoted to `int`.
```java
byte a = 10, b = 20;
// byte c = a + b;  → ERROR! Result is int
int c = a + b;      // OK
```

### Q40: What are literals in Java?
**A:** Fixed values in code.
```java
int dec = 100;       // decimal
int hex = 0xFF;      // hexadecimal
int oct = 077;       // octal
int bin = 0b1010;    // binary (Java 7+)
long l = 100L;       // long literal
double d = 1.5e3;    // scientific (1500.0)
int million = 1_000_000; // underscore separator (Java 7+)
```

### Q41: Can we use underscore in numeric literals?
**A:** Yes (Java 7+). `int x = 1_000_000;` → improves readability. Can't place at start, end, or next to decimal point.

## OPERATORS

### Q42: List all operator types in Java.
**A:** Arithmetic (+, -, *, /, %), Assignment (=, +=, etc.), Comparison (==, !=, >, <, >=, <=), Logical (&&, ||, !), Bitwise (&, |, ^, ~, <<, >>, >>>), Ternary (?:), instanceof, Unary (++, --, +, -, !, ~).

### Q43: What is the difference between `++i` and `i++`?
**A:**
```java
int i = 5;
System.out.println(i++);  // 5 (use then increment)
System.out.println(i);    // 6
System.out.println(++i);  // 7 (increment then use)
```

### Q44: What is short-circuit evaluation?
**A:** `&&` and `||` skip evaluating the second operand if the result is already determined.
```java
if (obj != null && obj.length() > 0) // if obj is null, length() is NOT called
```

### Q45: Difference between `&` and `&&`?
**A:** `&` evaluates BOTH sides always (bitwise AND). `&&` short-circuits (logical AND — skips right side if left is false).

### Q46: What is the ternary operator?
**A:** Shorthand for if-else: `condition ? valueIfTrue : valueIfFalse`
```java
String result = (age >= 18) ? "Adult" : "Minor";
```

### Q47: What is `instanceof`?
**A:** Checks if an object is an instance of a specific class or interface.
```java
String s = "Hello";
System.out.println(s instanceof String);  // true
System.out.println(s instanceof Object);  // true
```

### Q48: What are bitwise operators?
**A:**
```java
int a = 5, b = 3;  // a=0101, b=0011
a & b;   // 0001 = 1  (AND)
a | b;   // 0111 = 7  (OR)
a ^ b;   // 0110 = 6  (XOR)
~a;      // ...1010 = -6 (NOT)
a << 1;  // 1010 = 10 (left shift = multiply by 2)
a >> 1;  // 0010 = 2  (right shift = divide by 2)
```

### Q49: What is the difference between `>>` and `>>>`?
**A:** `>>` is signed right shift (preserves sign bit). `>>>` is unsigned right shift (fills with 0).

### Q50: What is operator precedence?
**A:** Order: Postfix (i++) → Unary (!x) → Arithmetic (*,/,% then +,-) → Relational (<,>) → Equality (==,!=) → Bitwise → Logical (&&, ||) → Ternary → Assignment. Use parentheses to be safe.

### Q51: Can we use `+` with Strings?
**A:** Yes. `+` concatenates strings.
```java
"Hello" + " " + "World"    // "Hello World"
"Age: " + 25               // "Age: 25" (int auto-converted)
10 + 20 + "Hello"          // "30Hello" (left to right)
"Hello" + 10 + 20          // "Hello1020" (string concat mode)
```

### Q52: What is the output of `10 + 20 + "30"`?
**A:** `"3030"`. First `10 + 20 = 30` (int addition), then `30 + "30" = "3030"` (string concat).

### Q53: What is the output of `"10" + 20 + 30`?
**A:** `"102030"`. `"10" + 20 = "1020"` (string), then `"1020" + 30 = "102030"`.

### Q54: What is the modulus operator?
**A:** `%` gives the remainder. `10 % 3 = 1`. Useful for even/odd check: `n % 2 == 0`.

### Q55: What happens when dividing int by zero?
**A:** `ArithmeticException`. But `double / 0` gives `Infinity` (no exception).
```java
// int x = 10 / 0;     → ArithmeticException
double d = 10.0 / 0;   // Infinity
double n = 0.0 / 0;    // NaN (Not a Number)
```

## CONDITIONAL STATEMENTS

### Q56: What are the types of conditional statements?
**A:** `if`, `if-else`, `if-else-if` ladder, nested if, `switch`.

### Q57: Can we have `if` without braces?
**A:** Yes, but only the next single statement is part of it. Always use braces to avoid bugs.
```java
if (true) System.out.println("One line OK");
```

### Q58: What is a nested if?
**A:**
```java
if (age >= 18) {
    if (hasID) {
        System.out.println("Entry allowed");
    }
}
```

### Q59: What types does switch support?
**A:** `byte`, `short`, `int`, `char`, `String` (Java 7+), `enum`, wrapper types. NOT `long`, `float`, `double`, `boolean`.

### Q60: What happens if we miss `break` in switch?
**A:** Fall-through — all subsequent cases execute until a `break` is found.

### Q61: What is the enhanced switch (Java 14+)?
**A:**
```java
String result = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    default -> "Other";
};  // no break needed, no fall-through
```

### Q62: Can switch have `null` case?
**A:** Not before Java 21. In Java 21+, `case null ->` is allowed.

### Q63: Can we use expressions in switch case labels?
**A:** No. Case labels must be **compile-time constants** (literals, final variables, enum constants).

### Q64: What is `switch` with `yield` (Java 14)?
**A:**
```java
int result = switch (x) {
    case 1 -> 10;
    case 2 -> {
        int val = 20 + 5;
        yield val;  // return value from block
    }
    default -> 0;
};
```

## LOOPS

### Q65: What are the types of loops in Java?
**A:** `for`, `while`, `do-while`, `for-each` (enhanced for).

### Q66: Difference between `for` and `for-each`?
**A:**
```java
// for → index control, can modify
for (int i = 0; i < arr.length; i++) { arr[i] = i * 2; }

// for-each → simpler, read-only, no index
for (int val : arr) { System.out.println(val); }
```

### Q67: When is `do-while` preferred over `while`?
**A:** When the loop body must execute **at least once** (e.g., menu-driven programs, input validation).
```java
int choice;
do {
    System.out.println("1. Add  2. Delete  3. Exit");
    choice = sc.nextInt();
} while (choice != 3);
```

### Q68: Can we have an infinite loop?
**A:** Yes.
```java
while (true) { }
for (;;) { }
do { } while (true);
```

### Q69: What is a labeled loop?
**A:** A loop with a name, used with `break`/`continue` for nested loops.
```java
outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) break outer;  // breaks BOTH loops
    }
}
```

### Q70: Difference between `break` and `continue`?
**A:** `break` exits the entire loop. `continue` skips current iteration and goes to next.

### Q71: Difference between `break` and `return`?
**A:** `break` exits the loop. `return` exits the entire method.

### Q72: Can `for` loop have empty sections?
**A:** Yes. `for(;;)` is valid (infinite loop). Each section (init, condition, update) is optional.

### Q73: Can we declare multiple variables in for loop?
**A:** Yes, if same type: `for (int i = 0, j = 10; i < j; i++, j--)`.

### Q74: What is the output?
```java
for (int i = 0; i < 5; i++);
System.out.println(i);
```
**A:** Compile error! `i` is not accessible outside the for loop. Also the for has empty body due to `;`.

### Q75: What is a nested loop?
**A:** A loop inside another loop. Used for 2D structures, patterns, matrix operations.
```java
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        System.out.print(i * j + " ");
    }
    System.out.println();
}
// 1 2 3
// 2 4 6
// 3 6 9
```

## ARRAYS

### Q76: What is an array?
**A:** Fixed-size container holding multiple values of the same type. Stored in contiguous memory.

### Q77: How to declare and initialize arrays?
**A:**
```java
int[] a = new int[5];          // default [0,0,0,0,0]
int[] b = {1, 2, 3, 4, 5};    // literal
int[] c = new int[]{1, 2, 3}; // anonymous array
```

### Q78: What is the default value of array elements?
**A:** `int` → 0, `double` → 0.0, `boolean` → false, `Object` → null.

### Q79: Can we change array size after creation?
**A:** No. Arrays are fixed-size. Use `ArrayList` for dynamic sizing.

### Q80: What is `ArrayIndexOutOfBoundsException`?
**A:** Thrown when accessing an invalid index (negative or >= length).

### Q81: What is the difference between `length` and `length()`?
**A:** `length` is a **property** of arrays. `length()` is a **method** of String. `size()` is a method of Collections.

### Q82: How to copy an array?
**A:**
```java
int[] original = {1, 2, 3, 4, 5};
int[] copy1 = Arrays.copyOf(original, 3);       // [1, 2, 3]
int[] copy2 = Arrays.copyOfRange(original, 1, 4); // [2, 3, 4]
int[] copy3 = original.clone();                   // [1, 2, 3, 4, 5]
System.arraycopy(original, 0, dest, 0, 3);       // system level copy
```

### Q83: How to sort an array?
**A:**
```java
int[] arr = {5, 2, 8, 1};
Arrays.sort(arr);                    // [1, 2, 5, 8]
Arrays.sort(arr, 0, 3);             // sort only index 0-2

String[] names = {"Charlie", "Alice", "Bob"};
Arrays.sort(names);                  // [Alice, Bob, Charlie]
Arrays.sort(names, Collections.reverseOrder()); // descending
```

### Q84: How to search in an array?
**A:**
```java
int[] arr = {1, 2, 5, 8};  // must be sorted for binarySearch
int idx = Arrays.binarySearch(arr, 5); // returns index
```

### Q85: How to compare two arrays?
**A:**
```java
int[] a = {1, 2, 3};
int[] b = {1, 2, 3};
System.out.println(a == b);              // false (different refs)
System.out.println(Arrays.equals(a, b)); // true (same content)
// For 2D arrays: Arrays.deepEquals(a2d, b2d);
```

### Q86: What is a 2D array?
**A:** Array of arrays (matrix).
```java
int[][] matrix = {{1,2,3}, {4,5,6}, {7,8,9}};
System.out.println(matrix[1][2]); // 6
System.out.println(matrix.length);    // 3 (rows)
System.out.println(matrix[0].length); // 3 (cols)
```

### Q87: What is a jagged array?
**A:** A 2D array where rows have different lengths.
```java
int[][] jagged = new int[3][];
jagged[0] = new int[]{1, 2};
jagged[1] = new int[]{3, 4, 5};
jagged[2] = new int[]{6};
```

### Q88: How to print an array?
**A:**
```java
int[] arr = {1, 2, 3};
System.out.println(Arrays.toString(arr));      // [1, 2, 3]
int[][] matrix = {{1,2},{3,4}};
System.out.println(Arrays.deepToString(matrix)); // [[1,2],[3,4]]
```

### Q89: Can we have an array of size 0?
**A:** Yes. `int[] arr = new int[0];` is valid. `arr.length` is 0.

### Q90: Are arrays objects in Java?
**A:** Yes. Arrays are objects that extend `Object`. They have `length` property and inherit `clone()`, `toString()`, etc.

### Q91: What is `ArrayStoreException`?
**A:** Thrown when storing wrong type in an object array.
```java
Object[] arr = new String[3];
arr[0] = 123;  // ArrayStoreException! Expected String
```

### Q92: How to convert array to ArrayList and vice versa?
**A:**
```java
// Array → List
String[] arr = {"A", "B", "C"};
List<String> list = Arrays.asList(arr);           // fixed-size list
List<String> list2 = new ArrayList<>(Arrays.asList(arr)); // mutable

// List → Array
String[] back = list.toArray(new String[0]);
```

### Q93: What is `Arrays.fill()`?
**A:** Fills array with a value.
```java
int[] arr = new int[5];
Arrays.fill(arr, 7);  // [7, 7, 7, 7, 7]
```

### Q94: How to find min/max in array without sorting?
**A:**
```java
int[] arr = {5, 2, 8, 1, 9};
int max = arr[0], min = arr[0];
for (int n : arr) {
    if (n > max) max = n;
    if (n < min) min = n;
}
// Or with streams: Arrays.stream(arr).max().getAsInt();
```

### Q95: How to reverse an array?
**A:**
```java
int[] arr = {1, 2, 3, 4, 5};
for (int i = 0; i < arr.length / 2; i++) {
    int temp = arr[i];
    arr[i] = arr[arr.length - 1 - i];
    arr[arr.length - 1 - i] = temp;
}
// [5, 4, 3, 2, 1]
```

### Q96: How to remove duplicates from an array?
**A:**
```java
int[] arr = {1, 2, 2, 3, 3, 4};
int[] unique = Arrays.stream(arr).distinct().toArray(); // [1, 2, 3, 4]
```

### Q97: What is the difference between `Arrays.asList()` and `List.of()`?
**A:**
| Feature | `Arrays.asList()` | `List.of()` (Java 9+) |
|---------|-------------------|----------------------|
| Mutable elements | Yes (set) | No |
| Add/Remove | No | No |
| Null elements | Yes | No |
| Backed by array | Yes | No |

### Q98: How to check if array contains a value?
**A:**
```java
int[] arr = {1, 2, 3, 4, 5};
boolean contains = Arrays.stream(arr).anyMatch(n -> n == 3); // true

String[] names = {"A", "B", "C"};
boolean has = Arrays.asList(names).contains("B"); // true
```

### Q99: What is `Arrays.mismatch()` (Java 9)?
**A:** Returns the index of the first mismatch between two arrays, or -1 if identical.
```java
int[] a = {1, 2, 3, 4};
int[] b = {1, 2, 5, 4};
System.out.println(Arrays.mismatch(a, b)); // 2
```

### Q100: What is `Arrays.stream()`?
**A:** Converts an array to a Stream for functional operations.
```java
int[] arr = {5, 3, 8, 1};
int sum = Arrays.stream(arr).sum();           // 17
double avg = Arrays.stream(arr).average().orElse(0); // 4.25
```

### Q101: What is a String?
**A:** An object of `java.lang.String` class representing a sequence of characters. Strings are **immutable**.

### Q102: Why is String immutable?
**A:** For String Pool sharing, security (passwords, URLs), thread safety, and hashCode caching.

### Q103: What is the String Pool?
**A:** A special memory area in heap where String literals are stored and shared. Two variables with same literal point to same object.

### Q104: How many objects in `String s = new String("Hello");`?
**A:** **2** — one in String Pool ("Hello" literal) and one in heap (new String).

### Q105: How many objects in `String s = "Hello";`?
**A:** **1** (in String Pool). If "Hello" already exists in pool, no new object — just a reference.

### Q106: What is `intern()` method?
**A:** Puts string into String Pool. Returns the pooled reference.
```java
String a = new String("Hello");
String b = a.intern();
String c = "Hello";
System.out.println(b == c); // true (both from pool)
```

### Q107: Difference between `==` and `equals()` for strings?
**A:** `==` compares references (memory address). `equals()` compares content.
```java
String a = "Hello", b = "Hello", c = new String("Hello");
a == b;        // true  (pool)
a == c;        // false (different objects)
a.equals(c);   // true  (same content)
```

### Q108: What is the difference between `equals()` and `equalsIgnoreCase()`?
**A:** `equals()` is case-sensitive. `equalsIgnoreCase()` ignores case.
```java
"hello".equals("Hello");           // false
"hello".equalsIgnoreCase("Hello"); // true
```

### Q109: How to get a character at a specific position?
**A:** `str.charAt(index)` — 0-based index.
```java
"Hello".charAt(0);  // 'H'
"Hello".charAt(4);  // 'o'
```

### Q110: How to get a substring?
**A:**
```java
"Hello World".substring(6);      // "World"
"Hello World".substring(0, 5);   // "Hello" (start inclusive, end exclusive)
```

### Q111: What is `indexOf()` and `lastIndexOf()`?
**A:**
```java
"Hello World Hello".indexOf("Hello");     // 0 (first occurrence)
"Hello World Hello".lastIndexOf("Hello"); // 12 (last occurrence)
"Hello".indexOf("xyz");                   // -1 (not found)
```

### Q112: How to check if string starts/ends with something?
**A:**
```java
"Hello World".startsWith("Hello"); // true
"Hello World".endsWith("World");   // true
```

### Q123: What is `concat()` method?

```java
// Same as "Hello" + " World"
```

### Q124: Difference between `concat()` and `+`?
**A:** `concat()` throws NullPointerException if argument is null. `+` converts null to "null" string. `+` uses StringBuilder internally.

### Q125: What is `format()` method?

### Q128: What is `matches()` method?
**A:** Checks if string matches a regex pattern.

### Q129: What is StringBuilder?
**A:** Mutable string. Best for frequent modifications. Not thread-safe.
```java
StringBuilder sb = new StringBuilder("Hello");
sb.append(" World");     // "Hello World"
sb.insert(5, ",");       // "Hello, World"
sb.delete(5, 6);         // "Hello World"

```

**A:** Same as StringBuilder but **thread-safe** (synchronized). Slightly slower.

### Q131: String vs StringBuilder vs StringBuffer?
**A:**
| Feature | String | StringBuilder | StringBuffer |
|---------|--------|--------------|-------------|
| Mutable | No | Yes | Yes |
| Thread-safe | Yes (immutable) | No | Yes |

### Q132: Why is String concatenation in loop bad?

```java
// BAD

for (int i = 0; i < 10000; i++) s += i; // 10000 objects!

// GOOD
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 10000; i++) sb.append(i); // 1 object
```

## METHODS

### Q156: What is a method?
**A:** A reusable block of code that performs a task. Defined inside a class.

### Q157: What are the parts of a method?
**A:** Access modifier, return type, method name, parameters, method body.
```java
public int add(int a, int b) {
    return a + b;
}
```

### Q158: What is method overloading?
**A:** Same method name, different parameters (number, type, or order). Compile-time polymorphism.
```java
int add(int a, int b)         { return a + b; }
double add(double a, double b){ return a + b; }
int add(int a, int b, int c)  { return a + b + c; }
```

### Q159: Can we overload by changing only return type?
**A:** No. Compile error. Parameters must differ.

### Q160: Is Java pass-by-value or pass-by-reference?
**A:** Always **pass-by-value**. For objects, the reference value is copied (not the object).

### Q161: What is recursion?
**A:** A method calling itself. Must have a base case.
```java
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
```

### Q162: What is the risk of recursion?
**A:** `StackOverflowError` if no base case or recursion is too deep.

### Q163: What are varargs?
**A:** Variable arguments — accept any number of parameters. Must be last parameter.
```java
int sum(int... nums) {
    int total = 0;
    for (int n : nums) total += n;
    return total;
}
sum(1, 2);        // 3
sum(1, 2, 3, 4);  // 10
```

### Q164: Can varargs be used with other parameters?
**A:** Yes, but varargs must be the LAST parameter.
```java
void greet(String name, int... scores) { }
```

### Q165: What is method signature?
**A:** Method name + parameter list. Does NOT include return type or access modifier.

## CLASSES & OBJECTS

### Q166: What is a class vs object?
**A:** Class = blueprint. Object = instance created from the blueprint with `new`.

### Q167: What is a constructor?
**A:** Special method to initialize objects. Same name as class, no return type.

### Q168: Types of constructors?
**A:** Default (no-arg), Parameterized, Copy constructor (manual in Java).

### Q169: What is the default constructor?
**A:** If no constructor is written, Java provides one with no parameters and empty body. If you write ANY constructor, the default is NOT provided.

### Q170: What is constructor overloading?
**A:** Multiple constructors with different parameters.
```java
class Student {
    Student() { }
    Student(String name) { }
    Student(String name, int age) { }
}
```

### Q171: Can constructor be `private`?
**A:** Yes. Used in Singleton Pattern to prevent external instantiation.

### Q172: Can constructor be `final`, `static`, or `abstract`?
**A:** No to all. Constructors can't be final, static, or abstract.

### Q173: What is constructor chaining?
**A:** Calling one constructor from another using `this()` or `super()`.
```java
class Item {
    Item() { this("Default"); }
    Item(String name) { this(name, 0); }
    Item(String name, int qty) { /* ... */ }
}
```

### Q174: Can `this()` and `super()` both be in same constructor?
**A:** No. Both must be the first statement, so only one is allowed.

### Q175: What is `this` keyword?
**A:** Refers to current object. Uses: resolve field/parameter conflict, call other constructors, pass/return current object.

### Q176: What is `super` keyword?
**A:** Refers to parent class. Uses: call parent constructor, access parent field/method.

### Q177: What is an Anonymous Object?
**A:** An object created without a reference variable. Used once.
```java
new Student().display(); // anonymous — can't reuse
```

### Q178: What is Object class?
**A:** Root of all Java classes. Every class implicitly extends `Object`.

### Q179: List methods of Object class.
**A:** `toString()`, `equals()`, `hashCode()`, `clone()`, `getClass()`, `finalize()`, `wait()`, `notify()`, `notifyAll()`.

### Q180: Why override `toString()`?
**A:** Default returns `ClassName@hexHashCode` which is not useful. Override for meaningful output.
```java
@Override
public String toString() {
    return "Student{name=" + name + ", age=" + age + "}";
}
```

### Q181: Why override `equals()` and `hashCode()` together?
**A:** Contract: equal objects must have same hashCode. Required for HashMap, HashSet to work correctly.

### Q182: What is `getClass()`?
**A:** Returns runtime class of the object. `obj.getClass().getName()`.

### Q183: What is `clone()` method?
**A:** Creates a copy of the object. Class must implement `Cloneable`.
```java
class Item implements Cloneable {
    @Override protected Object clone() throws CloneNotSupportedException {
        return super.clone();
    }
}
```

### Q184: Shallow copy vs Deep copy?
**A:** Shallow: copies references (shared inner objects). Deep: copies everything (new inner objects).

### Q185: What is a POJO?
**A:** Plain Old Java Object — a simple class with fields, getters, setters, no special restrictions.

### Q186: What is a Bean?
**A:** A Java class with: private fields, public getters/setters, no-arg constructor, implements Serializable.

### Q187: What is a DTO?
**A:** Data Transfer Object — a class used only to transfer data between layers. No business logic.

## ENCAPSULATION

### Q188: What is encapsulation?
**A:** Wrapping data + methods + restricting direct access using `private` fields and public getters/setters.

### Q189: What are access modifiers?
**A:** `private` (same class only), `default` (same package), `protected` (same package + subclass), `public` (everywhere).

### Q190: Can a class be `private`?
**A:** Only inner classes can be `private`. Top-level classes can only be `public` or `default`.

### Q191: Can a class be `protected`?
**A:** Only inner classes. Top-level classes cannot be `protected`.

### Q192: What are getters and setters?
**A:** Methods to read (`get`) and write (`set`) private fields with validation.

## INHERITANCE

### Q193: What is inheritance?
**A:** A class acquiring properties/methods of another class using `extends`. Promotes code reuse.

### Q194: Types of inheritance in Java?
**A:** Single, Multilevel, Hierarchical. NOT Multiple (with classes) or Hybrid.

### Q195: Why no multiple inheritance with classes?
**A:** Diamond Problem — ambiguity when two parents have same method. Solved using interfaces.

### Q196: What is `extends`?
**A:** Keyword for class inheritance: `class Dog extends Animal { }`.

### Q197: Can a class extend multiple classes?
**A:** No. Java supports only single inheritance with classes. Use interfaces for multiple.

### Q198: Can an interface extend another interface?
**A:** Yes. And an interface can extend **multiple** interfaces.

### Q199: What is `super()`?
**A:** Calls parent class constructor. Must be first statement. Added automatically if not written.

### Q200: What is method hiding?
**A:** When child class defines a static method with same signature as parent. It **hides** (not overrides) the parent's method.

### Q201: What is `IS-A` and `HAS-A` relationship?
**A:** IS-A = inheritance (`Dog IS-A Animal`). HAS-A = composition/aggregation (`Car HAS-A Engine`).

### Q202: What is composition vs aggregation?
**A:** Composition: strong ownership (Room can't exist without House). Aggregation: weak ownership (Teacher can exist without Department).

## POLYMORPHISM

### Q203: What is polymorphism?
**A:** One thing taking many forms. Compile-time (overloading) and Runtime (overriding).

### Q204: What is compile-time polymorphism?
**A:** Method overloading — same name, different params. Resolved at compile time.

### Q205: What is runtime polymorphism?
**A:** Method overriding — parent reference, child object. Method decided at runtime.
```java
Animal a = new Dog(); // upcasting
a.sound(); // calls Dog's sound() at runtime
```

### Q206: What is method overriding?
**A:** Child class redefines parent method with same name, same params, same or covariant return type.

### Q207: Rules of overriding?
**A:** Same name, same params, same/covariant return type. Access can't be more restrictive. Can't override `final`, `static`, `private` methods. Must use `@Override`.

### Q208: What is covariant return type?
**A:** Overriding method can return a subtype of the parent's return type.
```java
class Animal { Animal create() { return new Animal(); } }
class Dog extends Animal { Dog create() { return new Dog(); } } // covariant
```

### Q209: Can we override `private` methods?
**A:** No. Private methods aren't visible to child classes.

### Q210: Can we override `final` methods?
**A:** No. `final` prevents overriding.

### Q211: Can we override `static` methods?
**A:** No. Static methods are hidden (method hiding), not overridden.

### Q212: What is upcasting?
**A:** Parent reference pointing to child object. Automatic and safe.
```java
Animal a = new Dog(); // upcasting
```

### Q213: What is downcasting?
**A:** Casting parent reference back to child. Manual and risky.
```java
Dog d = (Dog) a; // may throw ClassCastException if wrong type
```

### Q214: What is dynamic method dispatch?
**A:** JVM determines which overridden method to call at runtime based on the actual object type, not reference type.

---

## ABSTRACTION

### Q215: What is abstraction?
**A:** Showing WHAT something does, hiding HOW. Achieved via abstract classes and interfaces.

### Q216: What is an abstract class?
**A:** A class with `abstract` keyword. Can have abstract methods (no body) and concrete methods. Cannot be instantiated.

### Q217: Can abstract class have constructor?
**A:** Yes. Called when child class is instantiated using `super()`.

### Q218: Can abstract class have non-abstract methods?
**A:** Yes. Can have both abstract and concrete methods.

### Q219: Can we make abstract class without abstract method?
**A:** Yes. Just prevents instantiation.

### Q220: What is an interface?
**A:** A contract defining methods a class must implement. Uses `implements`. Supports multiple inheritance.

### Q221: What was added to interfaces in Java 8?
**A:** `default` methods (with body) and `static` methods.

### Q222: What was added to interfaces in Java 9?
**A:** `private` methods (helper methods within interface).

### Q223: Abstract class vs Interface?
**A:**
| Feature | Abstract Class | Interface |
|---------|---------------|-----------|
| Methods | Abstract + concrete | Abstract + default + static + private |
| Fields | Any | Only public static final |
| Constructor | Yes | No |
| Inheritance | Single | Multiple |

### Q224: When to use abstract class vs interface?
**A:** Abstract class for shared code among related classes. Interface for contracts across unrelated classes or multiple inheritance.

### Q225: What is a marker interface?
**A:** An empty interface with no methods (e.g., `Serializable`, `Cloneable`). Acts as a tag to the JVM.

## IMPORTANT KEYWORDS

### Q226: What is `static` keyword?
**A:** Belongs to class, not object. Used for: variables (shared), methods (no object needed), blocks (class loading), inner classes.

### Q227: Can we access non-static from static context?
**A:** No. Static methods can't access instance variables/methods directly. Need an object.

### Q228: What is a static block?
**A:** Runs once when class is loaded. Used for static initialization.
```java
static { dbUrl = "jdbc:mysql://localhost/mydb"; }
```

### Q229: Execution order of static block, instance block, constructor?
**A:** Static block → Instance block → Constructor. Static block runs only once.

### Q230: What is `final` keyword?
**A:** Variable → constant. Method → can't override. Class → can't extend.

### Q231: Can a `final` variable be initialized later?
**A:** Yes, only once. Blank final variable must be initialized in constructor.

### Q232: What is a `final` method?
**A:** Cannot be overridden by child classes.

### Q233: What is a `final` class?
**A:** Cannot be inherited. Example: `String`, `Integer`, `Math`.

### Q234: What is `final` parameter?
**A:** Parameter can't be reassigned inside the method.
```java
void method(final int x) {
    // x = 10;  → ERROR
}
```

### Q235: Difference between `final`, `finally`, `finalize()`?
**A:**
- `final`: constant / prevent override / prevent inheritance
- `finally`: block that always executes after try-catch
- `finalize()`: called by GC before destroying object (deprecated Java 9+)

### Q236: What is `transient` keyword?
**A:** Excludes a field from serialization.
```java
transient String password; // not saved to file
```

### Q237: What is `volatile` keyword?
**A:** Variable is always read from main memory, not thread cache. Guarantees visibility across threads.

### Q238: What is `synchronized` keyword?
**A:** Only one thread can access the method/block at a time.

### Q239: What is `native` keyword?
**A:** Method implemented in C/C++ (platform-specific). Used in JNI.

### Q240: What is `strictfp` keyword?
**A:** Ensures floating-point calculations give same result on all platforms. Deprecated since Java 17.

### Q241: What is `assert` keyword?
**A:** For debugging. Throws `AssertionError` if condition is false.
```java
assert age > 0 : "Age must be positive";
```

## EXCEPTION HANDLING

### Q242: What is an exception?
**A:** An unwanted event that disrupts normal program flow.

### Q243: Exception hierarchy?
**A:**
```
Throwable
├── Error (unrecoverable)
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   └── VirtualMachineError
└── Exception (recoverable)
    ├── Checked (compile-time)
    │   ├── IOException
    │   ├── SQLException
    │   ├── FileNotFoundException
    │   └── ClassNotFoundException
    └── Unchecked (runtime)
        ├── NullPointerException
        ├── ArithmeticException
        ├── ArrayIndexOutOfBoundsException
        ├── ClassCastException
        ├── NumberFormatException
        └── IllegalArgumentException
```

### Q244: Checked vs Unchecked exceptions?
**A:** Checked: must handle (compile-time), extends Exception. Unchecked: optional handling (runtime), extends RuntimeException.

### Q245: What is Error vs Exception?
**A:** Error: serious, unrecoverable (OutOfMemoryError). Exception: recoverable, should handle.

### Q246: List 10 common exceptions.
**A:**
1. `NullPointerException` — calling method on null
2. `ArithmeticException` — divide by zero
3. `ArrayIndexOutOfBoundsException` — invalid array index
4. `ClassCastException` — wrong type cast
5. `NumberFormatException` — invalid parseInt
6. `StringIndexOutOfBoundsException` — invalid string index
7. `IllegalArgumentException` — wrong method argument
8. `IOException` — input/output failure
9. `FileNotFoundException` — file doesn't exist
10. `StackOverflowError` — infinite recursion

### Q247: How does try-catch work?
**A:**
```java
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Error: " + e.getMessage());
}
```

### Q248: Can we have multiple catch blocks?
**A:** Yes. Order: specific to general.
```java
try { }
catch (FileNotFoundException e) { }  // specific first
catch (IOException e) { }
catch (Exception e) { }              // most general last
```

### Q249: What is multi-catch (Java 7)?
**A:** Catching multiple exceptions in one catch block.
```java
catch (IOException | SQLException e) { }
```

### Q250: What is `finally` block?
**A:** Always executes after try-catch, whether exception occurs or not. Used for cleanup.

### Q251: When does `finally` NOT execute?
**A:** Only when `System.exit()` is called, JVM crashes, or thread is killed.

### Q252: Can we have `try` without `catch`?
**A:** Yes, with `finally`: `try { } finally { }`.

### Q253: Can we have `try` without `catch` and `finally`?
**A:** No. `try` must have at least `catch` or `finally`.

### Q254: What if both `catch` and `finally` have return?
**A:** `finally`'s return overrides `catch`'s return. Avoid `return` in `finally`.

### Q255: What is `throw` keyword?
**A:** Explicitly throw an exception.
```java
throw new IllegalArgumentException("Invalid input!");
```

### Q256: What is `throws` keyword?
**A:** Declares that a method may throw exceptions.
```java
void readFile() throws IOException, FileNotFoundException { }
```

### Q257: Difference between `throw` and `throws`?
**A:** `throw` — actually throws an exception (inside method body). `throws` — declares possibility (in method signature).

### Q258: Can we throw checked exception without declaring?
**A:** No. Checked exceptions must be declared with `throws` or caught.

### Q259: Can we throw unchecked exception without declaring?
**A:** Yes. Unchecked exceptions don't need `throws` declaration.

### Q260: What is exception propagation?
**A:** If method doesn't handle exception, it passes to calling method, up the call stack, until handled or JVM terminates.

### Q261: How do checked vs unchecked propagate?
**A:** Checked: must be declared at each level. Unchecked: propagates automatically.

### Q262: What is `e.getMessage()` vs `e.printStackTrace()`?
**A:** `getMessage()` returns short error message. `printStackTrace()` prints full stack trace.

### Q263: What is custom exception?
**A:** User-defined exception extending `Exception` (checked) or `RuntimeException` (unchecked).
```java
class InsufficientFundsException extends Exception {
    InsufficientFundsException(String msg) { super(msg); }
}
```

### Q264: What is try-with-resources (Java 7)?
**A:** Auto-closes resources implementing `AutoCloseable`.
```java
try (BufferedReader br = new BufferedReader(new FileReader("f.txt"))) {
    br.readLine();
} // br automatically closed
```

### Q265: Can try-with-resources have multiple resources?
**A:** Yes. Separated by semicolons.
```java
try (FileReader fr = new FileReader("f.txt");
     BufferedReader br = new BufferedReader(fr)) { }
```

### Q266: What is suppressed exception?
**A:** When exception in `try` block AND exception in `close()` both occur, the close exception is suppressed. Access via `e.getSuppressed()`.

### Q267: Can we rethrow an exception?
**A:** Yes.
```java
catch (Exception e) {
    throw e;  // rethrow
    // or wrap: throw new CustomException("msg", e);
}
```

### Q268: What is exception chaining?
**A:** Wrapping one exception inside another to preserve original cause.
```java
throw new RuntimeException("Processing failed", originalException);
```

### Q269: Can constructor throw exception?
**A:** Yes. Both checked and unchecked.

### Q270: Can we catch `Error`?
**A:** Technically yes, but should NOT. Errors are serious and usually unrecoverable.

### Q271: What is `StackOverflowError`?
**A:** Too deep recursion or infinite method calls. Stack memory exhausted.

### Q272: What is `OutOfMemoryError`?
**A:** Heap memory exhausted. Too many objects or memory leak.

### Q273: What is `ClassNotFoundException` vs `NoClassDefFoundError`?
**A:** `ClassNotFoundException`: checked exception when loading class dynamically. `NoClassDefFoundError`: class was present at compile time but missing at runtime.

### Q274: What is `NullPointerException`?
**A:** Calling method/accessing field on a `null` reference.
```java
String s = null;
s.length(); // NullPointerException!
```

### Q275: How to avoid NullPointerException?
**A:** Null checks, `Optional`, `Objects.requireNonNull()`, use empty collections instead of null.

## WRAPPER CLASSES & ENUMS

### Q276: What are wrapper classes?
**A:** Object versions of primitives. Needed for collections.
`int→Integer, double→Double, char→Character, boolean→Boolean`, etc.

### Q277: What is autoboxing and unboxing?
**A:** Autoboxing: `int → Integer` auto. Unboxing: `Integer → int` auto.

### Q278: What is Integer cache?
**A:** Java caches Integer -128 to 127. Same-value integers in this range are same object.
```java
Integer a = 127, b = 127;
a == b;  // true (cached)
Integer c = 128, d = 128;
c == d;  // false (not cached!)
```

### Q279: How to convert String to int?
**A:** `Integer.parseInt("123")` or `Integer.valueOf("123")`.

### Q280: Difference between `parseInt()` and `valueOf()`?
**A:** `parseInt()` returns primitive `int`. `valueOf()` returns `Integer` object.

### Q281: What is `NumberFormatException`?
**A:** Thrown when parsing invalid string to number: `Integer.parseInt("abc")`.

### Q282: What is an enum?
**A:** Special class for fixed set of constants.
```java
enum Season { SPRING, SUMMER, AUTUMN, WINTER }
```

### Q283: Can enum have constructor?
**A:** Yes, but always `private`. Can't use `new` with enum.

### Q284: Can enum have methods and fields?
**A:** Yes.
```java
enum Planet {
    EARTH(5.97e24);
    private final double mass;
    Planet(double mass) { this.mass = mass; }
    double getMass() { return mass; }
}
```

### Q285: Can enum implement interface?
**A:** Yes. Each constant can have its own implementation.

### Q286: Can enum extend a class?
**A:** No. Enum implicitly extends `java.lang.Enum`.

### Q287: What is `values()` and `valueOf()` in enum?
**A:** `values()` returns array of all constants. `valueOf("NAME")` returns the constant.
```java
for (Season s : Season.values()) System.out.println(s);
Season s = Season.valueOf("SUMMER");
```

### Q288: What is `ordinal()` in enum?
**A:** Returns the position (0-based) of the constant.

### Q289: Can we use enum in switch?
**A:** Yes.
```java
switch (season) {
    case SUMMER -> System.out.println("Hot");
    case WINTER -> System.out.println("Cold");
}
```

### Q290: What is `EnumSet`?
**A:** Efficient Set implementation for enums, uses bit-vector internally. `EnumSet.of(Season.SUMMER, Season.WINTER)`.

### Q291: What is `EnumMap`?
**A:** Efficient Map with enum keys. `new EnumMap<>(Season.class)`.

## COLLECTIONS OVERVIEW

### Q301: What is the Collections Framework?
**A:** A unified architecture for representing and manipulating groups of objects. Located in `java.util`. Includes interfaces, implementations, and algorithms.

### Q302: What is the Collection hierarchy?
**A:**
```
Iterable
└── Collection
    ├── List (ordered, allows duplicates)
    │   ├── ArrayList
    │   ├── LinkedList
    │   └── Vector → Stack
    ├── Set (no duplicates)
    │   ├── HashSet
    │   ├── LinkedHashSet
    │   └── TreeSet (sorted)
    └── Queue
        ├── LinkedList
        ├── PriorityQueue
        └── Deque → ArrayDeque

Map (separate hierarchy)
├── HashMap
├── LinkedHashMap
├── TreeMap (sorted)
├── Hashtable
└── ConcurrentHashMap
```

### Q303: Difference between Collection and Collections?
**A:** `Collection` = interface (root of hierarchy). `Collections` = utility class with static helper methods (sort, reverse, etc.).

### Q304: What is `Iterable` interface?
**A:** Root interface. Provides `iterator()` method. Enables for-each loop.

### Q305: What is an Iterator?
**A:** Object to traverse collections one element at a time.
```java
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if (s.equals("remove")) it.remove();
}
```

### Q306: Difference between Iterator and ListIterator?
**A:** Iterator: forward only, works on all Collections. ListIterator: bidirectional, only for List, can add/set elements.

### Q307: What is fail-fast vs fail-safe?
**A:** **Fail-fast** (ArrayList, HashMap): throws `ConcurrentModificationException` if modified during iteration. **Fail-safe** (CopyOnWriteArrayList, ConcurrentHashMap): works on clone, no exception.

### Q308: What is `Comparable` interface?
**A:** Defines natural ordering. Implement `compareTo()` method.
```java
class Student implements Comparable<Student> {
    int age;
    public int compareTo(Student other) {
        return this.age - other.age; // ascending
    }
}
Collections.sort(students); // uses compareTo
```

### Q309: What is `Comparator` interface?
**A:** External comparison for custom ordering.
```java
list.sort(Comparator.comparing(Student::getName));
list.sort(Comparator.comparing(Student::getAge).reversed());
list.sort(Comparator.comparing(Student::getName)
    .thenComparing(Student::getAge));
```

### Q310: Comparable vs Comparator?
**A:**
| Feature | Comparable | Comparator |
|---------|-----------|------------|
| Package | java.lang | java.util |
| Method | compareTo() | compare() |
| Where | Inside class | Outside class |
| Count | Single ordering | Multiple orderings |
| Modifies class | Yes | No |

## LIST

### Q311: What is ArrayList?
**A:** Resizable array. Random access O(1). Add/remove from end O(1), from middle O(n).
```java
List<String> list = new ArrayList<>();
list.add("A");
list.get(0);     // "A"
list.set(0, "B");
list.remove(0);
list.size();
```

### Q312: What is the default capacity of ArrayList?
**A:** **10**. Grows by 50% when full (new capacity = old * 1.5).

### Q313: How does ArrayList grow internally?
**A:** When full, creates new array with 50% more capacity, copies old elements, replaces internal array.

### Q314: How to create ArrayList with specific capacity?
**A:** `new ArrayList<>(100)` — avoids resizing if you know the size.

### Q315: What is LinkedList?
**A:** Doubly-linked list. Implements List and Deque. Insert/delete O(1) at known position. Access O(n).
```java
LinkedList<String> ll = new LinkedList<>();
ll.addFirst("A"); ll.addLast("B");
ll.getFirst(); ll.getLast();
ll.removeFirst(); ll.removeLast();
```

### Q316: ArrayList vs LinkedList?
**A:**
| Feature | ArrayList | LinkedList |
|---------|-----------|------------|
| Internal | Dynamic array | Doubly linked list |
| Access | O(1) (fast) | O(n) (slow) |
| Insert/Delete | O(n) (shift) | O(1) (pointer change) |
| Memory | Less (no pointers) | More (prev/next pointers) |
| Use when | Frequent reads | Frequent insert/delete |

### Q317: What is Vector?
**A:** Synchronized version of ArrayList. Thread-safe but slow. Legacy class.

### Q318: ArrayList vs Vector?
**A:** ArrayList: not synchronized, faster, grows by 50%. Vector: synchronized, slower, grows by 100%.

### Q319: What is Stack?
**A:** LIFO (Last In, First Out). Extends Vector.
```java
Stack<Integer> stack = new Stack<>();
stack.push(1); stack.push(2);
stack.peek();  // 2 (top, no remove)
stack.pop();   // 2 (top, remove)
```

### Q320: What is a better alternative to Stack?
**A:** `ArrayDeque`. Stack is legacy and synchronized. ArrayDeque is faster.

### Q321: How to convert List to array and vice versa?
**A:**
```java
List<String> list = Arrays.asList("A", "B", "C");  // fixed-size
List<String> mutable = new ArrayList<>(list);       // mutable
String[] arr = list.toArray(new String[0]);         // List → array
List<String> list2 = List.of("A", "B");             // immutable (Java 9)
```

### Q322: What is `List.of()` and `List.copyOf()` (Java 9+)?
**A:** Creates immutable lists. `List.of("A", "B")` — unmodifiable. `List.copyOf(existingList)` — immutable copy.

### Q323: How to sort a List?
**A:**
```java
Collections.sort(list);                          // natural order
Collections.sort(list, Comparator.reverseOrder()); // reverse
list.sort(Comparator.naturalOrder());            // Java 8+
```

### Q324: How to make an ArrayList read-only?
**A:** `List<String> readOnly = Collections.unmodifiableList(list);`

### Q325: How to synchronize an ArrayList?
**A:** `List<String> syncList = Collections.synchronizedList(new ArrayList<>());`

### Q326: What is `subList()`?
**A:** Returns a view of a portion. Changes reflect in original list.
```java
List<Integer> sub = list.subList(1, 4); // index 1 to 3
```

### Q327: How to remove elements while iterating?
**A:**
```java
// Using Iterator
Iterator<String> it = list.iterator();
while (it.hasNext()) { if (it.next().equals("X")) it.remove(); }

// Using removeIf (Java 8)
list.removeIf(s -> s.equals("X"));
```

### Q328: What is `CopyOnWriteArrayList`?
**A:** Thread-safe variant. Creates new copy of array on every write. Read is fast, write is expensive. Fail-safe iteration.

## SET

### Q329: What is a Set?
**A:** Collection with **no duplicates**. Unordered (HashSet), ordered (LinkedHashSet), sorted (TreeSet).

### Q330: How does HashSet check for duplicates?
**A:** Uses `hashCode()` first, then `equals()`. If both match, it's a duplicate.

### Q331: What is HashSet internally?
**A:** Backed by `HashMap`. Elements stored as keys with a dummy value.

### Q332: What is LinkedHashSet?
**A:** Maintains **insertion order** + no duplicates. Slightly slower than HashSet.

### Q333: What is TreeSet?
**A:** Sorted set using Red-Black tree. O(log n) operations. Elements must be Comparable or use Comparator.
```java
TreeSet<Integer> ts = new TreeSet<>();
ts.add(30); ts.add(10); ts.add(20);
System.out.println(ts); // [10, 20, 30]
ts.first(); // 10
ts.last();  // 30
```

### Q334: HashSet vs LinkedHashSet vs TreeSet?
**A:**
| Feature | HashSet | LinkedHashSet | TreeSet |
|---------|---------|--------------|---------|
| Order | None | Insertion | Sorted |
| Null | 1 null | 1 null | No null |
| Performance | O(1) | O(1) | O(log n) |
| Internal | HashMap | LinkedHashMap | Red-Black Tree |

### Q335: Can we add `null` to TreeSet?
**A:** No. `TreeSet` uses `compareTo()` which throws `NullPointerException` on null.

### Q336: What is `NavigableSet`?
**A:** Interface with navigation methods. TreeSet implements it.
```java
TreeSet<Integer> ts = new TreeSet<>(List.of(10,20,30,40));
ts.floor(25);    // 20 (greatest <= 25)
ts.ceiling(25);  // 30 (smallest >= 25)
ts.lower(20);    // 10 (greatest < 20)
ts.higher(20);   // 30 (smallest > 20)
ts.headSet(25);  // [10, 20]
ts.tailSet(25);  // [30, 40]
```

### Q337: How to find union, intersection, difference of Sets?
**A:**
```java
Set<Integer> a = new HashSet<>(Set.of(1,2,3));
Set<Integer> b = Set.of(2,3,4);
a.addAll(b);     // Union: [1,2,3,4]
a.retainAll(b);  // Intersection: [2,3]
a.removeAll(b);  // Difference: [1]
```

### Q338: What is `Set.of()` (Java 9)?
**A:** Creates an immutable Set. No nulls, no duplicates.
```java
Set<String> s = Set.of("A", "B", "C");
```

## MAP

### Q339: What is Map?
**A:** Key-value pair collection. Keys are unique. Not part of Collection interface.

### Q340: What is HashMap?
**A:** Hash table based Map. O(1) get/put. Allows 1 null key, multiple null values. Not ordered.

### Q341: How does HashMap work internally?
**A:** Uses array of buckets. Key's `hashCode()` determines bucket index. Collision handled by linked list (Java 7) or tree (Java 8+, when bucket size > 8).

### Q342: What is the default capacity and load factor of HashMap?
**A:** Default capacity = **16**, load factor = **0.75**. Rehashes when 75% full (threshold = 12).

### Q343: What is rehashing?
**A:** When Map reaches threshold, capacity doubles and all entries are redistributed to new buckets.

### Q344: What happens on HashMap collision?
**A:** Multiple keys with same bucket index → stored in linked list. Java 8+: if list grows > 8 entries, converts to balanced tree (O(log n)).

### Q345: Why override `hashCode()` and `equals()` for HashMap keys?
**A:** HashMap uses `hashCode()` to find bucket and `equals()` to find exact key. Without proper overrides, identical keys may not match.

### Q346: Rules for good `hashCode()`?
**A:** Equal objects MUST have equal hashCode. Same fields used in `equals()` should be used in `hashCode()`. Use `Objects.hash(field1, field2)`.

### Q347: Can we use mutable object as HashMap key?
**A:** Can, but SHOULDN'T. If key changes after insertion, hashCode changes, and you can't find the entry anymore.

### Q348: Common HashMap methods?
**A:**
```java
Map<String, Integer> map = new HashMap<>();
map.put("A", 1);          // insert
map.get("A");             // 1
map.getOrDefault("B", 0); // 0 (key absent)
map.containsKey("A");     // true
map.containsValue(1);     // true
map.remove("A");          // remove
map.size();               // count
map.isEmpty();            // boolean
map.putIfAbsent("C", 3); // add only if key missing
map.keySet();             // Set of keys
map.values();             // Collection of values
map.entrySet();           // Set of Map.Entry
```

### Q349: How to iterate a HashMap?
**A:**
```java
// for-each with entrySet (BEST)
for (Map.Entry<String, Integer> e : map.entrySet()) {
    System.out.println(e.getKey() + "=" + e.getValue());
}
// forEach (Java 8)
map.forEach((k, v) -> System.out.println(k + "=" + v));
```

### Q350: What is LinkedHashMap?
**A:** HashMap + maintains **insertion order** (or access order). Slightly slower than HashMap.
```java
Map<String, Integer> map = new LinkedHashMap<>();
```

### Q351: What is TreeMap?
**A:** Sorted Map using Red-Black tree. Keys sorted naturally or by Comparator. O(log n) operations.
```java
TreeMap<String, Integer> tm = new TreeMap<>();
tm.put("Banana", 2); tm.put("Apple", 1); tm.put("Cherry", 3);
tm.firstKey();     // "Apple"
tm.lastKey();      // "Cherry"
tm.floorKey("B");  // "Banana" (greatest <= B)
```

### Q352: HashMap vs LinkedHashMap vs TreeMap?
**A:**
| Feature | HashMap | LinkedHashMap | TreeMap |
|---------|---------|--------------|---------|
| Order | None | Insertion | Sorted (keys) |
| Null key | 1 | 1 | No |
| Performance | O(1) | O(1) | O(log n) |

### Q353: What is Hashtable?
**A:** Legacy synchronized map. No null keys or values. Use `ConcurrentHashMap` instead.

### Q354: HashMap vs Hashtable?
**A:** HashMap: not synchronized, allows null key/values, faster. Hashtable: synchronized, no nulls, slower, legacy.

### Q355: What is `ConcurrentHashMap`?
**A:** Thread-safe HashMap without locking entire map. Uses segment-level locking (Java 7) / CAS + synchronized blocks (Java 8+). Better than Hashtable.

### Q356: What is `Map.of()` and `Map.ofEntries()` (Java 9)?
**A:**
```java
Map<String, Integer> m = Map.of("A", 1, "B", 2);  // up to 10 pairs
Map<String, Integer> m2 = Map.ofEntries(
    Map.entry("A", 1), Map.entry("B", 2)
);
// Both are immutable, no null keys/values
```

### Q357: What is `compute()`, `merge()` and `computeIfAbsent()`?
**A:**
```java
// computeIfAbsent: add value if key missing
map.computeIfAbsent("key", k -> new ArrayList<>()).add("value");

// compute: transform existing value
map.compute("count", (k, v) -> (v == null) ? 1 : v + 1);

// merge: combine old and new value
map.merge("count", 1, Integer::sum); // increment by 1
```

### Q358: How to find the most frequent element using Map?
**A:**
```java
int[] arr = {1, 2, 2, 3, 3, 3};
Map<Integer, Long> freq = Arrays.stream(arr).boxed()
    .collect(Collectors.groupingBy(n -> n, Collectors.counting()));
// {1=1, 2=2, 3=3}
```

### Q359: How to sort HashMap by value?
**A:**
```java
map.entrySet().stream()
    .sorted(Map.Entry.comparingByValue())
    .forEach(System.out::println);
```

### Q360: What is `WeakHashMap`?
**A:** Map where keys are weakly referenced. GC can collect entries when keys have no strong references. Used for caches.

## QUEUE & DEQUE

### Q361: What is Queue?
**A:** FIFO (First In, First Out) collection.
```java
Queue<String> q = new LinkedList<>();
q.offer("A"); q.offer("B");
q.peek();    // "A" (front, no remove)
q.poll();    // "A" (front, remove)
```

### Q362: Difference between `add()` and `offer()`?
**A:** `add()` throws exception if full. `offer()` returns false if full.

### Q363: Difference between `remove()` and `poll()`?
**A:** `remove()` throws exception if empty. `poll()` returns null if empty.

### Q364: Difference between `element()` and `peek()`?
**A:** `element()` throws exception if empty. `peek()` returns null if empty.

### Q365: What is PriorityQueue?
**A:** Elements ordered by priority (natural order or Comparator). Not FIFO.
```java
PriorityQueue<Integer> pq = new PriorityQueue<>(); // min-heap
pq.offer(30); pq.offer(10); pq.offer(20);
pq.poll(); // 10 (smallest first)

// Max-heap:
PriorityQueue<Integer> maxPQ = new PriorityQueue<>(Comparator.reverseOrder());
```

### Q366: What is the internal structure of PriorityQueue?
**A:** Binary heap (min-heap by default). Backed by array. O(log n) insert/remove, O(1) peek.

### Q367: What is Deque?
**A:** Double-Ended Queue — insert/remove from both ends.
```java
Deque<String> dq = new ArrayDeque<>();
dq.offerFirst("A"); dq.offerLast("B");
dq.peekFirst();  dq.peekLast();
dq.pollFirst();  dq.pollLast();
```

### Q368: What is ArrayDeque?
**A:** Resizable array implementation of Deque. Faster than LinkedList for stack/queue operations.

### Q369: ArrayDeque vs LinkedList for Queue?
**A:** ArrayDeque: no null elements, faster (no node allocation), uses array. LinkedList: allows null, uses linked nodes.

### Q370: How to implement Stack using Deque?
**A:**
```java
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1); stack.push(2); // LIFO
stack.peek();  // 2
stack.pop();   // 2
```

## GENERICS

### Q381: What is Generics?
**A:** Type parameters for classes/methods. Provides compile-time type safety.
```java
List<String> list = new ArrayList<>(); // only Strings allowed
```

### Q382: What problem does Generics solve?
**A:** Eliminates ClassCastException at runtime by catching type errors at compile time.

### Q383: What is a generic class?
**A:**
```java
class Box<T> {
    private T value;
    void set(T val) { this.value = val; }
    T get() { return value; }
}
Box<String> sb = new Box<>(); sb.set("Hello");
Box<Integer> ib = new Box<>(); ib.set(42);
```

### Q384: What is a generic method?
**A:**
```java
<T> void printArray(T[] arr) {
    for (T elem : arr) System.out.println(elem);
}
```

### Q385: What are bounded type parameters?
**A:**
```java
<T extends Number> double sum(List<T> list) {
    return list.stream().mapToDouble(Number::doubleValue).sum();
}
// T must be Number or its subclass
```

### Q386: What are wildcards?
**A:**
- `?` → unknown type
- `? extends T` → upper bound (T or subtypes) — read only
- `? super T` → lower bound (T or supertypes) — write only

### Q387: What is PECS?
**A:** Producer Extends, Consumer Super. `? extends T` for reading. `? super T` for writing.

### Q388: What is type erasure?
**A:** Generics are compile-time only. At runtime, all generic types are replaced with `Object` (or bound). `List<String>` becomes `List` at runtime.

### Q389: Can we create generic array?
**A:** No. `new T[]` is illegal due to type erasure. Use `ArrayList<T>` instead.

### Q390: Can primitives be used with generics?
**A:** No. Use wrappers: `List<Integer>`, not `List<int>`.

### Q391: What is diamond operator?
**A:** `<>` infers type from left side. `List<String> list = new ArrayList<>()`.

### Q392: What is raw type?
**A:** Using generic class without type parameter: `List list = new ArrayList()`. Unsafe, no compile-time check.

### Q393: What is the difference between `List<?>` and `List<Object>`?
**A:** `List<?>` accepts any `List<X>`. `List<Object>` only accepts `List<Object>`.

### Q394: Can we overload methods by generic type?
**A:** No. Due to type erasure, `method(List<String>)` and `method(List<Integer>)` have same erasure.

### Q395: What are multiple bounds?
**A:** `<T extends Comparable<T> & Serializable>` — T must implement both.

### Q396: What is a generic interface?
**A:**
```java
interface Pair<K, V> {
    K getKey();
    V getValue();
}
```

### Q397: What is a recursive type bound?
**A:** `<T extends Comparable<T>>` — T must be comparable to itself. Common for sorting.

### Q398: What is `@SuppressWarnings("unchecked")`?
**A:** Suppresses compiler warnings about unchecked generic casts.

### Q399: Can static methods use class type parameters?
**A:** No. Static methods don't have access to instance type parameters. Must declare their own: `static <T> void method(T t)`.

### Q400: What is reifiable vs non-reifiable type?
**A:** Reifiable: type info available at runtime (primitives, raw types, `List<?>`). Non-reifiable: erased at runtime (`List<String>`, `List<T>`).

## LAMBDA EXPRESSIONS

### Q401: What is a Lambda expression?
**A:** Short anonymous function. Syntax: `(params) -> expression` or `(params) -> { statements }`.
```java
(a, b) -> a + b
name -> name.toUpperCase()
() -> System.out.println("Hello")
```

### Q402: What is a Functional Interface?
**A:** Interface with exactly ONE abstract method. Can have default/static methods. Annotated with `@FunctionalInterface`.

### Q403: List built-in functional interfaces?
**A:**
| Interface | Method | Input → Output |
|-----------|--------|----------------|
| `Predicate<T>` | `test(T)` | T → boolean |
| `Function<T,R>` | `apply(T)` | T → R |
| `Consumer<T>` | `accept(T)` | T → void |
| `Supplier<T>` | `get()` | () → T |
| `UnaryOperator<T>` | `apply(T)` | T → T |
| `BinaryOperator<T>` | `apply(T,T)` | (T,T) → T |
| `BiFunction<T,U,R>` | `apply(T,U)` | (T,U) → R |
| `BiPredicate<T,U>` | `test(T,U)` | (T,U) → boolean |
| `BiConsumer<T,U>` | `accept(T,U)` | (T,U) → void |

### Q404: Example of Predicate?
**A:**
```java
Predicate<Integer> isEven = n -> n % 2 == 0;
isEven.test(4);  // true
isEven.test(5);  // false

// Chaining
Predicate<Integer> isPositive = n -> n > 0;
Predicate<Integer> isPositiveEven = isEven.and(isPositive);
```

### Q405: Example of Function?
**A:**
```java
Function<String, Integer> length = String::length;
length.apply("Hello"); // 5

Function<Integer, Integer> doubleIt = n -> n * 2;
Function<Integer, Integer> addTen = n -> n + 10;
doubleIt.andThen(addTen).apply(5); // (5*2)+10 = 20
doubleIt.compose(addTen).apply(5); // (5+10)*2 = 30
```

### Q406: Example of Consumer?
**A:**
```java
Consumer<String> printer = System.out::println;
printer.accept("Hello"); // prints "Hello"

Consumer<String> shout = s -> System.out.println(s.toUpperCase());
printer.andThen(shout).accept("hi"); // prints "hi" then "HI"
```

### Q407: Example of Supplier?
**A:**
```java
Supplier<Double> random = Math::random;
random.get(); // 0.73481...

Supplier<List<String>> listFactory = ArrayList::new;
List<String> newList = listFactory.get();
```

### Q408: What is a method reference?
**A:** Shorthand for lambda that calls existing method.
```java
// Static method: ClassName::method
Function<String, Integer> parse = Integer::parseInt;

// Instance method: object::method
String str = "Hello";
Supplier<Integer> len = str::length;

// Instance method of any object: ClassName::method
Function<String, String> upper = String::toUpperCase;

// Constructor: ClassName::new
Supplier<ArrayList<String>> factory = ArrayList::new;
```

### Q409: Types of method references?
**A:** Static (`Math::abs`), Instance of specific object (`str::length`), Instance of arbitrary object (`String::toLowerCase`), Constructor (`ArrayList::new`).

### Q410: Can lambda access local variables?
**A:** Yes, but they must be **effectively final** (not reassigned after initialization).
```java
int x = 10;
Runnable r = () -> System.out.println(x); // OK, x is effectively final
// x = 20; → would make lambda illegal
```

## STREAMS

### Q411: What is a Stream?
**A:** A sequence of elements from a source, supporting aggregate operations. NOT a data structure — processes data pipeline.
```java
List<String> result = names.stream()
    .filter(n -> n.startsWith("A"))
    .map(String::toUpperCase)
    .sorted()
    .collect(Collectors.toList());
```

### Q412: How to create a Stream?
**A:**
```java
list.stream();                      // from Collection
Arrays.stream(array);              // from Array
Stream.of("A", "B", "C");         // from values
Stream.empty();                     // empty stream
Stream.generate(Math::random);     // infinite (Supplier)
Stream.iterate(0, n -> n + 2);    // infinite: 0, 2, 4, 6...
Stream.iterate(0, n -> n < 10, n -> n + 2); // bounded (Java 9)
```

### Q413: What are intermediate operations?
**A:** Return a new Stream. Lazy — not executed until terminal operation.
`filter`, `map`, `flatMap`, `sorted`, `distinct`, `peek`, `limit`, `skip`, `mapToInt/Long/Double`.

### Q414: What are terminal operations?
**A:** Produce result or side-effect. Triggers processing.
`forEach`, `collect`, `count`, `reduce`, `min`, `max`, `findFirst`, `findAny`, `anyMatch`, `allMatch`, `noneMatch`, `toArray`.

### Q415: What is `filter()`?
**A:** Keeps elements matching the predicate.
```java
List<Integer> evens = numbers.stream().filter(n -> n % 2 == 0).toList();
```

### Q416: What is `map()`?
**A:** Transforms each element.
```java
List<String> upper = names.stream().map(String::toUpperCase).toList();
```

### Q417: What is `flatMap()`?
**A:** Transforms and flattens nested collections into single stream.
```java
List<List<Integer>> nested = List.of(List.of(1,2), List.of(3,4));
List<Integer> flat = nested.stream().flatMap(Collection::stream).toList();
// [1, 2, 3, 4]
```

### Q418: What is `reduce()`?
**A:** Combines all elements to single value.
```java
int sum = numbers.stream().reduce(0, Integer::sum);
Optional<Integer> max = numbers.stream().reduce(Integer::max);
String concat = words.stream().reduce("", (a, b) -> a + " " + b);
```

### Q419: What is `collect()`?
**A:** Collects stream into a collection or value.
```java
List<String> list = stream.collect(Collectors.toList());
Set<String> set = stream.collect(Collectors.toSet());
String joined = stream.collect(Collectors.joining(", "));
Map<String, Integer> map = stream.collect(
    Collectors.toMap(s -> s, String::length));
```

### Q420: What is `Collectors.groupingBy()`?
**A:** Groups elements by a classifier.
```java
Map<Integer, List<String>> byLength = names.stream()
    .collect(Collectors.groupingBy(String::length));
// {4=[John, Jane], 5=[Alice]}
```

### Q421: What is `Collectors.partitioningBy()`?
**A:** Splits into two groups (true/false).
```java
Map<Boolean, List<Integer>> parts = numbers.stream()
    .collect(Collectors.partitioningBy(n -> n % 2 == 0));
// {false=[1,3,5], true=[2,4,6]}
```

### Q422: What is `Collectors.counting()` and `Collectors.summarizing`?
**A:**
```java
long count = stream.collect(Collectors.counting());
IntSummaryStatistics stats = numbers.stream()
    .collect(Collectors.summarizingInt(Integer::intValue));
stats.getSum(); stats.getAverage(); stats.getMax(); stats.getMin();
```

### Q423: What is `sorted()` in Stream?
**A:**
```java
names.stream().sorted().toList();                         // natural
names.stream().sorted(Comparator.reverseOrder()).toList(); // reverse
students.stream().sorted(Comparator.comparing(Student::getAge)).toList();
```

### Q424: What is `distinct()`?
**A:** Removes duplicates (uses `equals()`).
```java
List<Integer> unique = List.of(1,2,2,3,3).stream().distinct().toList();
// [1, 2, 3]
```

### Q425: What is `limit()` and `skip()`?
**A:**
```java
Stream.of(1,2,3,4,5).limit(3).toList();  // [1, 2, 3]
Stream.of(1,2,3,4,5).skip(2).toList();   // [3, 4, 5]
```

### Q426: What is `peek()`?
**A:** Performs action without consuming stream. For debugging.
```java
list.stream()
    .peek(System.out::println)  // prints each element
    .filter(n -> n > 3)
    .collect(Collectors.toList());
```

### Q427: What is parallel stream?
**A:** Processes elements in parallel using ForkJoinPool.
```java
list.parallelStream().filter(...).collect(...);
// or list.stream().parallel()...
```

### Q428: When to use parallel stream?
**A:** Large data sets, independent operations, CPU-intensive tasks. NOT for I/O, small data, or ordered operations.

### Q429: What is `findFirst()` vs `findAny()`?
**A:** `findFirst()` returns first element (ordered). `findAny()` returns any element (faster in parallel).

### Q430: What is `anyMatch()`, `allMatch()`, `noneMatch()`?
**A:**
```java
List<Integer> nums = List.of(1, 2, 3, 4, 5);
nums.stream().anyMatch(n -> n > 4);   // true (at least one)
nums.stream().allMatch(n -> n > 0);   // true (every one)
nums.stream().noneMatch(n -> n > 10); // true (none)
```

### Q431: Stream vs Collection?
**A:** Collection stores data. Stream processes data. Streams are lazy, can't be reused, don't modify source.

### Q432: Can we reuse a Stream?
**A:** No. Once consumed by terminal operation, stream is closed. Create new stream.

### Q433: What are primitive streams?
**A:** `IntStream`, `LongStream`, `DoubleStream` — avoid boxing overhead.
```java
IntStream.range(1, 5);      // 1, 2, 3, 4
IntStream.rangeClosed(1, 5); // 1, 2, 3, 4, 5
IntStream.of(1, 2, 3).sum(); // 6
```

### Q434: How to convert stream to primitive stream?
**A:**
```java
IntStream intStream = list.stream().mapToInt(Integer::intValue);
Stream<Integer> boxed = intStream.boxed();
```

## OPTIONAL

### Q435: What is Optional?
**A:** Container that may or may not have a value. Avoids `NullPointerException`.
```java
Optional<String> opt = Optional.of("Hello");
Optional<String> empty = Optional.empty();
Optional<String> nullable = Optional.ofNullable(null);
```

### Q436: How to use Optional?
**A:**
```java
opt.isPresent();          // true
opt.isEmpty();            // false (Java 11)
opt.get();                // "Hello" (throws if empty)
opt.orElse("default");    // "Hello" (or give default)
opt.orElseGet(() -> "default"); // Lazy default
opt.orElseThrow();        // throws NoSuchElementException
opt.orElseThrow(() -> new RuntimeException("Not found"));
opt.ifPresent(System.out::println); // prints if present
opt.map(String::toUpperCase);       // Optional["HELLO"]
opt.filter(s -> s.length() > 3);    // Optional["Hello"]
opt.flatMap(s -> Optional.of(s.toLowerCase())); // avoids Optional<Optional>
```

### Q437: `orElse()` vs `orElseGet()`?
**A:** `orElse()` always evaluates default. `orElseGet()` only evaluates if empty (lazy). Use `orElseGet()` for expensive defaults.

### Q438: When NOT to use Optional?
**A:** As method parameters, class fields, collection values, or serialized objects. Use for return types only.

### Q456: What is a Thread?
**A:** The smallest unit of execution within a process. Java supports multithreading — running multiple threads simultaneously.

### Q457: What is a Process vs Thread?
**A:** Process: independent program with own memory. Thread: lightweight sub-unit of process, shares memory with other threads.

### Q458: How to create a thread?
**A:**
```java
// Way 1: Extend Thread
class MyThread extends Thread {
    public void run() { System.out.println("Running"); }
}
new MyThread().start();

// Way 2: Implement Runnable (preferred)
class MyRunnable implements Runnable {
    public void run() { System.out.println("Running"); }
}
new Thread(new MyRunnable()).start();

// Way 3: Lambda
new Thread(() -> System.out.println("Running")).start();
```

### Q459: Difference between `start()` and `run()`?
**A:** `start()` creates new thread and calls `run()`. `run()` directly runs in current thread (no new thread).

### Q460: Can we call `start()` twice?
**A:** No. Throws `IllegalThreadStateException`.

### Q461: What are thread states?
**A:** `NEW` → `RUNNABLE` → `BLOCKED` / `WAITING` / `TIMED_WAITING` → `TERMINATED`.

### Q462: What is `Thread.sleep()`?
**A:** Pauses current thread for specified milliseconds. Doesn't release locks.
```java
Thread.sleep(1000); // sleep 1 second
```

### Q463: What is `Thread.yield()`?
**A:** Hints scheduler to give other threads a turn. Not guaranteed.

### Q464: What is `join()`?
**A:** Waits for a thread to finish.
```java
Thread t = new Thread(() -> System.out.println("Done"));
t.start();
t.join(); // wait for t to finish
```

### Q465: What is thread priority?
**A:** 1 (MIN) to 10 (MAX), default 5. `t.setPriority(Thread.MAX_PRIORITY)`. Hint only.

### Q466: What is a daemon thread?
**A:** Background thread that doesn't prevent JVM from exiting. Example: GC. `t.setDaemon(true)`.

### Q467: User thread vs Daemon thread?
**A:** User threads: JVM waits for them to finish. Daemon threads: JVM exits even if they're running.

### Q468: What is thread safety?
**A:** Code that works correctly when accessed by multiple threads simultaneously. Achieved via synchronization, volatile, concurrent classes.

### Q469: What is `synchronized` keyword?
**A:** Allows only one thread at a time to execute a method/block.
```java
synchronized void increment() { count++; }

// or block
synchronized (this) { count++; }
```

### Q470: What is a monitor/lock?
**A:** Every object has an intrinsic lock (monitor). `synchronized` acquires the lock. Only one thread can hold it.

### Q471: What is `static synchronized` method?
**A:** Lock is on the Class object, not instance. Only one thread across all instances.

### Q472: What is a race condition?
**A:** Bug when multiple threads access shared data and the result depends on execution order.

### Q473: What is a deadlock?
**A:** Two threads each waiting for the other's lock. Both blocked forever.
```java
// Thread 1: locks A, waits for B
// Thread 2: locks B, waits for A  → DEADLOCK
```

### Q474: How to prevent deadlock?
**A:** Lock ordering (always acquire locks in same order), timeouts, `tryLock()`, avoid nested locks.

### Q475: What is `volatile`?
**A:** Guarantees visibility — variable is always read from main memory, not thread cache. Doesn't guarantee atomicity.
```java
volatile boolean running = true;
```

### Q476: What is `wait()`, `notify()`, `notifyAll()`?
**A:** Inter-thread communication. Must be called inside `synchronized`.
```java
synchronized (lock) {
    while (!condition) lock.wait();    // release lock, wait
}
synchronized (lock) {
    condition = true;
    lock.notify();     // wake one waiting thread
    lock.notifyAll();  // wake ALL waiting threads
}
```

### Q477: `wait()` vs `sleep()`?
**A:** `wait()` releases lock, works with `notify()`, must be in synchronized. `sleep()` keeps lock, just pauses.

### Q478: What is `Callable` vs `Runnable`?
**A:** `Runnable.run()` returns void. `Callable.call()` returns a value and can throw exception.
```java
Callable<Integer> task = () -> { return 42; };
```

### Q479: What is `Future`?
**A:** Represents the result of an async computation.
```java
Future<Integer> future = executor.submit(() -> 42);
int result = future.get(); // blocks until done
future.isDone();
future.cancel(true);
```

### Q480: What is `ExecutorService`?
**A:** Thread pool manager. Reuses threads instead of creating new ones.
```java
ExecutorService exec = Executors.newFixedThreadPool(5);
exec.submit(() -> System.out.println("Task"));
exec.shutdown();
```

### Q481: Types of thread pools?
**A:**
```java
Executors.newFixedThreadPool(n);     // fixed n threads
Executors.newCachedThreadPool();     // creates threads as needed
Executors.newSingleThreadExecutor(); // single thread
Executors.newScheduledThreadPool(n); // scheduled tasks
```

### Q482: What is `ScheduledExecutorService`?
**A:** For scheduling tasks.
```java
ScheduledExecutorService ses = Executors.newScheduledThreadPool(2);
ses.schedule(() -> System.out.println("Delayed"), 5, TimeUnit.SECONDS);
ses.scheduleAtFixedRate(() -> System.out.println("Repeat"), 0, 2, TimeUnit.SECONDS);
```

### Q483: What is `ThreadLocal`?
**A:** Each thread gets its own copy of the variable.
```java
ThreadLocal<Integer> tl = ThreadLocal.withInitial(() -> 0);
tl.set(42);
tl.get(); // 42 (only in this thread)
```

### Q484: What is `ReentrantLock`?
**A:** Explicit lock with more features than `synchronized`.
```java
ReentrantLock lock = new ReentrantLock();
lock.lock();
try { /* critical section */ }
finally { lock.unlock(); }
```

### Q485: `ReentrantLock` vs `synchronized`?
**A:** ReentrantLock: tryLock, timed lock, interruptible, fair mode. synchronized: simpler, auto-release.

### Q486: What is `ReadWriteLock`?
**A:** Allows multiple readers OR one writer.
```java
ReadWriteLock rwl = new ReentrantReadWriteLock();
rwl.readLock().lock();   // multiple threads can read
rwl.writeLock().lock();  // exclusive write
```

### Q487: What is `CountDownLatch`?
**A:** Blocks until count reaches zero.
```java
CountDownLatch latch = new CountDownLatch(3);
// 3 threads each call: latch.countDown();
latch.await(); // waits until count = 0
```

### Q488: What is `CyclicBarrier`?
**A:** Threads wait at barrier until all arrive, then proceed together. Reusable.

### Q489: What is `Semaphore`?
**A:** Controls access to a resource by limiting number of concurrent threads.
```java
Semaphore sem = new Semaphore(3); // max 3 threads
sem.acquire(); // get permit
sem.release(); // return permit
```

### Q490: What is `AtomicInteger`?
**A:** Thread-safe integer operations without locks.
```java
AtomicInteger count = new AtomicInteger(0);
count.incrementAndGet();     // 1
count.compareAndSet(1, 5);   // true, now 5
```

### Q491: What is `CompletableFuture`?
**A:** Advanced async programming.
```java
CompletableFuture.supplyAsync(() -> fetchData())
    .thenApply(data -> process(data))
    .thenAccept(result -> save(result))
    .exceptionally(ex -> { log(ex); return null; });
```

### Q492: Key `CompletableFuture` methods?
**A:**
```java
supplyAsync(Supplier)  // async task with result
thenApply(Function)    // transform result
thenAccept(Consumer)   // consume result
thenCompose(Function)  // chain futures
thenCombine(CF, BiFn)  // combine two futures
allOf(cf1, cf2, cf3)   // wait for all
anyOf(cf1, cf2, cf3)   // wait for any
exceptionally(Function) // handle errors
```

### Q493: What is `ForkJoinPool`?
**A:** Thread pool designed for divide-and-conquer tasks. Work-stealing algorithm. Used by parallel streams.

### Q494: What is the Producer-Consumer pattern?
**A:** One thread produces data, another consumes. Uses `BlockingQueue`.
```java
BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(10);
// Producer: queue.put(item);
// Consumer: int item = queue.take();
```

### Q495: What is `CopyOnWriteArrayList`?
**A:** Thread-safe ArrayList. Copies array on every write. Good for read-heavy, write-rare scenarios.

### Q496: What is `ConcurrentHashMap` vs `Hashtable`?
**A:** ConcurrentHashMap: segment-level locking, allows concurrent reads, better performance. Hashtable: locks entire map, slower.

### Q497: What are `Atomic` classes?
**A:** `AtomicInteger`, `AtomicLong`, `AtomicBoolean`, `AtomicReference` — lock-free thread-safe operations using CAS (Compare-And-Swap).

### Q498: What is starvation?
**A:** A thread never gets CPU time because other threads monopolize it.

### Q499: What is livelock?
**A:** Threads keep responding to each other but make no progress (unlike deadlock where they're stuck).

### Q500: What is thread-safe singleton?
**A:**
```java
class Singleton {
    private static volatile Singleton instance;
    private Singleton() {}
    static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) instance = new Singleton();
            }
        }
        return instance;
    }
}
```

### Q501: What are I/O Streams in Java?
**A:** Byte-based (`InputStream`/`OutputStream`) and character-based (`Reader`/`Writer`) for reading/writing data.

### Q502: Byte stream vs Character stream?
**A:** Byte: for binary data (images, audio). Character: for text (automatic encoding).

### Q503: How to read a file?
**A:**
```java
// Modern way (Java 11+)
String content = Files.readString(Path.of("file.txt"));

// BufferedReader
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    String line;
    while ((line = br.readLine()) != null) System.out.println(line);
}

// All lines
List<String> lines = Files.readAllLines(Path.of("file.txt"));
```

### Q504: How to write to a file?
**A:**
```java
// Modern way
Files.writeString(Path.of("file.txt"), "Hello World");

// BufferedWriter
try (BufferedWriter bw = new BufferedWriter(new FileWriter("file.txt"))) {
    bw.write("Hello");
    bw.newLine();
}

// Append mode
Files.writeString(Path.of("f.txt"), "append", StandardOpenOption.APPEND);
```

### Q505: What is `File` class?
**A:** Represents file/directory path. Not used for reading/writing content.
```java
File f = new File("test.txt");
f.exists(); f.isFile(); f.isDirectory();
f.getName(); f.length(); f.delete();
f.mkdir();
```

### Q506: `File` vs `Path`?
**A:** `Path` (Java 7, `java.nio`) is modern replacement with more features. Use `Path` + `Files` utility class.

### Q507: What is `Files` utility class?
**A:**
```java
Files.exists(path);
Files.createFile(path);
Files.createDirectory(path);
Files.copy(source, target);
Files.move(source, target);
Files.delete(path);
Files.size(path);
Files.list(path);   // Stream<Path> of directory entries
Files.walk(path);   // recursive Stream<Path>
```

### Q508: What is try-with-resources for I/O?
**A:** Auto-closes resources. Any `AutoCloseable` object can be used.
```java
try (var br = new BufferedReader(new FileReader("f.txt"))) {
    // br auto-closed
}
```

### Q509: What is serialization?
**A:** Converting object to byte stream for storage/transmission. Class must implement `Serializable`.
```java
class Student implements Serializable {
    String name;
    transient String password; // excluded
}
```

### Q510: How to serialize and deserialize?
**A:**
```java
// Serialize
try (ObjectOutputStream oos = new ObjectOutputStream(
        new FileOutputStream("student.ser"))) {
    oos.writeObject(student);
}

// Deserialize
try (ObjectInputStream ois = new ObjectInputStream(
        new FileInputStream("student.ser"))) {
    Student s = (Student) ois.readObject();
}
```

### Q511: What is `serialVersionUID`?
**A:** Version identifier for serialized class. If class changes, mismatched UID causes `InvalidClassException`.
```java
private static final long serialVersionUID = 1L;
```

### Q512: What is `transient` keyword in serialization?
**A:** Fields marked `transient` are skipped during serialization. Default value on deserialization.

### Q513: What is `Externalizable`?
**A:** Interface for custom serialization control. Must implement `writeExternal()` and `readExternal()`.

### Q514: Can we serialize static fields?
**A:** No. Static fields belong to the class, not the object, and are not serialized.

### Q515: What is `Scanner` class?
**A:** For reading user input.
```java
Scanner sc = new Scanner(System.in);
String name = sc.nextLine();
int age = sc.nextInt();
sc.close();
```

### Q516: `Scanner` vs `BufferedReader`?
**A:** Scanner: parses input, slower, easier. BufferedReader: raw text, faster, for files/large input.

### Q517: How to read/write properties file?
**A:**
```java
Properties prop = new Properties();
prop.load(new FileInputStream("config.properties"));
String value = prop.getProperty("key");
prop.setProperty("key", "value");
prop.store(new FileOutputStream("config.properties"), "comment");
```

### Q518: What is NIO?
**A:** `java.nio` (New I/O) — non-blocking I/O with channels, buffers, selectors. Higher performance than `java.io`.

### Q519: What is `Channel` and `Buffer` in NIO?
**A:** Channel: bidirectional data connection. Buffer: container for data (ByteBuffer, CharBuffer). Data flows: Channel → Buffer → Program.

### Q520: How to walk a directory tree?
**A:**
```java
Files.walk(Path.of("src"))
    .filter(p -> p.toString().endsWith(".java"))
    .forEach(System.out::println);
```

### Q521: Reverse a number.
```java
int num = 12345, rev = 0;
while (num > 0) {
    rev = rev * 10 + num % 10;
    num /= 10;
}
```

### Q527: Find GCD (Greatest Common Divisor).
```java
int gcd(int a, int b) {

}
```

### Q530: Find second largest in array.

### Q538: Check balanced parentheses.
```java
boolean isBalanced(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(') stack.push(')');
        else if (c == '{') stack.push('}');
        else if (c == '[') stack.push(']');
        else if (stack.isEmpty() || stack.pop() != c) return false;
    }
    return stack.isEmpty();
}
```

### Q542: Merge two sorted arrays.
```java
int[] merge(int[] a, int[] b) {
    int[] result = new int[a.length + b.length];
    int i = 0, j = 0, k = 0;
    while (i < a.length && j < b.length)
        result[k++] = (a[i] <= b[j]) ? a[i++] : b[j++];
    while (i < a.length) result[k++] = a[i++];
    while (j < b.length) result[k++] = b[j++];
    return result;
}
```

### Q543: Binary Search.
```java
int binarySearch(int[] arr, int target) {
    int low = 0, high = arr.length - 1;
    while (low <= high) {
        int mid = low + (high - low) / 2;
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) low = mid + 1;
        else high = mid - 1;
    }
    return -1;
}
```

### Q544: Bubble Sort.
```java
void bubbleSort(int[] arr) {
    for (int i = 0; i < arr.length - 1; i++)
        for (int j = 0; j < arr.length - 1 - i; j++)
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j]; arr[j] = arr[j + 1]; arr[j + 1] = temp;
            }
}
```

### Q546: Convert Roman to Integer.
```java
int romanToInt(String s) {
    Map<Character, Integer> map = Map.of(
        'I',1, 'V',5, 'X',10, 'L',50, 'C',100, 'D',500, 'M',1000);
    int result = 0;
    for (int i = 0; i < s.length(); i++) {
        int val = map.get(s.charAt(i));
        if (i + 1 < s.length() && val < map.get(s.charAt(i + 1)))
            result -= val;
        else result += val;
    }
    return result;
}
```

### Q548: Find majority element (appears > n/2 times).
```java
// Boyer-Moore Voting Algorithm
int majorityElement(int[] nums) {
    int candidate = nums[0], count = 1;
    for (int i = 1; i < nums.length; i++) {
        if (count == 0) { candidate = nums[i]; count = 1; }
        else if (nums[i] == candidate) count++;
        else count--;
    }
    return candidate;
}
```

### Q550: Producer-Consumer using BlockingQueue.
```java
BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(5);

// Producer
new Thread(() -> {
    for (int i = 1; i <= 10; i++) {
        try { queue.put(i); System.out.println("Produced: " + i); }
        catch (InterruptedException e) { Thread.currentThread().interrupt(); }
    }
}).start();

// Consumer
new Thread(() -> {
    for (int i = 0; i < 10; i++) {
        try { System.out.println("Consumed: " + queue.take()); }
        catch (InterruptedException e) { Thread.currentThread().interrupt(); }
    }
}).start();
```
